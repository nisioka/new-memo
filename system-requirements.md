# 要件定義書（System Requirements Specification）

## 1. 機能要件
   ID	カテゴリ	要件内容	優先度
   F-1	認証	GitHub OAuth認証を実装する。ローカル利用モードもあり。	高
   F-2	メモ作成	新規Markdownファイルを作成可能。	高
   F-3	編集	シンプルなTextareaで編集。入力補完・Markdownプレビュー切替あり。	高
   F-4	保存／コミット	保存時にGitコミットを実行。自動コミットもサポート。	高
   F-5	履歴閲覧	コミット履歴を時系列で表示。	高
   F-6	差分表示	unified diff形式で2バージョンを比較可能。	高
   F-7	ブランチ管理	ブランチの作成・切替・マージ・削除をGUIで操作可能。	中
   F-8	オフライン編集	ローカルストレージでキャッシュし、再接続時に同期。	高
   F-9	ファイル管理	複数のMarkdownファイルを作成・切替可能。	高
   F-10	GitHub連携	各ファイル＝1リポジトリの構成。PATでpush/pullを行う。	高
   F-11	バックアップ	自動でローカルバックアップを保持。	中
   F-12	設定管理	コミットメッセージテンプレート・自動保存間隔などを設定可能。	低

## 2. 非機能要件
   項目	要件
   パフォーマンス	保存・差分表示は1秒以内（通常ファイルサイズ1MB未満前提）
   オフライン耐性	通信断時も編集継続可。ネット復帰後自動push。
   セキュリティ	PATは暗号化保存。通信はすべてHTTPS。
   ストレージ	ローカルキャッシュはIndexedDBまたはFileSystem APIを使用。
   操作性	スマホ縦画面で片手操作しやすいUI。ボタン数を最小化。
   信頼性	Git操作失敗時のロールバック機構を持つ。
   継続性	MarkdownとGitのみで完結するため将来もデータロックインなし。

## 3. 将来的な拡張想定（オプション）
   機能	概要
   AI連携（MCPサーバ対応）	AIが取得・更新できるインタフェースを提供。
   差分ベースAI補正	外部AIで修正→差分確認→部分マージ。
   CLIモード	同じリポジトリをCLIクライアントでも操作可能に。
